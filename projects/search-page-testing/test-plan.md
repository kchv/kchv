# Тест-план: страница поиска

## ID

`search-page-01`

## Цель тестирования

Оценить качество доработанной страницы поиска и проверить возможность релиза изменений.

## Объект тестирования

Страница поиска информационного веб-сервиса:

- search input;
- result list;
- tabs / sections;
- filters;
- sorting controls;
- pagination controls;
- direct URL parameters.

## Функции, подлежащие тестированию

- Ввод и обработка поисковых запросов.
- Пагинация результатов.
- Группировка результатов по разделам.
- Применение фильтров.
- Сортировка выдачи.
- Отображение результатов.
- Переходы из выдачи в материалы.

## Подход и стратегия

- Тип: ручное функциональное тестирование, UI/UX-тестирование, регрессионное тестирование.
- Методы: тестирование черного ящика, классы эквивалентности, граничные значения, переходы состояний, попарное тестирование, негативные проверки.
- Документация: тест-план, чек-лист, тест-кейсы, регрессионный чек-лист, баг-репорты.
- Приоритет: сначала smoke и основные пользовательские сценарии, затем фильтры, пагинация, граничные случаи и регрессия.

## Входит в тестирование

- Основная страница поиска.
- Разделы выдачи.
- Пагинация.
- Сортировка.
- Фильтры.
- URL-параметры.
- Негативные проверки поискового ввода.

## Не входит в тестирование

- Авторизация.
- Нагрузочное тестирование.
- Полная кроссбраузерная матрица.
- Полное мобильное тестирование.
- Проверка всех разрешений экрана.

## Критерии приемки

### Критерии успешного завершения

- Нет блокирующих и критических дефектов.
- Дефекты высокого приоритета не блокируют основной пользовательский сценарий.
- Основные пользовательские сценарии пройдены.
- Все найденные дефекты заведены в бэклог.

### Критерии неуспешного завершения

- Поиск недоступен.
- Переходы между разделами не работают.
- Пагинация ломает выдачу.
- Фильтры или сортировка приводят к некорректным данным.

## Риски

- Неполные требования.
- Изменение данных на окружении во время проверки.
- Ограничение отображения выдачи первыми страницами.
- Расхождение между данными на frontend и backend.

## Тестовые данные

| Тип данных | Пример | Цель |
| --- | --- | --- |
| Релевантный запрос | `НДС`, `декларация`, `увольнение` | Проверка выдачи и сортировки |
| Редкий запрос | Длинная точная фраза | Проверка пустых состояний в отдельных разделах |
| Невалидный ввод | `<script>`, `' OR 1=1 --`, пробелы | Проверка устойчивости и валидации |
| Длинный ввод | 500+ символов | Проверка ограничений поля и верстки |
