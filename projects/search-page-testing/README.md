# Пример тестирования страницы поиска

Публичный пример тестовой документации для доработанной страницы поиска информационного веб-сервиса.

В разделе собраны тест-план, чек-лист, тест-кейсы, регрессионные проверки и баг-репорты по функциональности поиска.

## Задача

Проверить доработанную страницу поиска перед релизом.

После изменений в продукте появились:

- пагинация результатов;
- группировка результатов по разделам;
- фильтры;
- сортировка;
- отдельные сценарии для разделов "Новости и статьи", "Форум", "Нормативные документы".

## Что проверялось

- Ввод и обработка поисковых запросов.
- Переходы между разделами выдачи.
- Пагинация и прямые переходы по URL-параметрам.
- Сортировка по релевантности и дате.
- Фильтры в разделе форума.
- Негативные сценарии: пустые запросы, спецсимволы, длинные строки, SQL-like ввод.
- Регрессионные проверки старого функционала поиска.

## Результат

- Подготовлен тест-план.
- Сформирован набор функциональных и регрессионных проверок.
- Проверены позитивные и негативные сценарии.
- Зафиксированы дефекты по сортировке и счетчикам результатов.
- Критические и блокирующие дефекты не обнаружены.

## Вывод

Качество релиза можно оценить как достаточное при условии фиксации найденных дефектов в бэклоге и мониторинга ключевых метрик после выкладки.
