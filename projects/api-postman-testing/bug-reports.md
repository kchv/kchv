# API баг-репорты

## API-BR-001: Сумма корзины не пересчитывается после изменения количества товара

| Поле | Значение |
| --- | --- |
| Серьезность | Major |
| Приоритет | Высокий |
| Endpoint | `PATCH /cart/items/{itemId}` |
| Окружение | Staging API |
| Статус | Открыт |

### Шаги воспроизведения

1. Добавить в корзину товар стоимостью `1200` в количестве `1`.
2. Отправить `PATCH /cart/items/{itemId}` с телом `{"quantity": 2}`.
3. Отправить `GET /cart`.

### Фактический результат

Количество товара изменилось на `2`, но итоговая сумма корзины осталась `1200`.

### Ожидаемый результат

Итоговая сумма корзины пересчитана и равна `2400`.

### Влияние

Некорректная сумма корзины может привести к неправильной сумме заказа и расхождению с платежом.

## API-BR-002: Endpoint товара возвращает 500 при невалидном формате id

| Поле | Значение |
| --- | --- |
| Серьезность | Major |
| Приоритет | Высокий |
| Endpoint | `GET /products/{id}` |
| Окружение | Staging API |
| Статус | Открыт |

### Шаги воспроизведения

1. Отправить `GET /products/abc@@@`.

### Фактический результат

API возвращает `500 Internal Server Error`.

### Ожидаемый результат

API возвращает `400 Bad Request` с кодом ошибки валидации `INVALID_PRODUCT_ID`.

### Влияние

Невалидный пользовательский ввод обрабатывается как серверная ошибка, что загрязняет мониторинг и скрывает проблему валидации.

## API-BR-003: Ответ 401 содержит внутренние детали исключения

| Поле | Значение |
| --- | --- |
| Серьезность | Minor |
| Приоритет | Средний |
| Endpoint | `GET /users/me` |
| Окружение | Staging API |
| Статус | Открыт |

### Шаги воспроизведения

1. Отправить `GET /users/me` без заголовка `Authorization`.

### Фактический результат

Ответ содержит внутреннее имя класса и фрагмент stack trace.

### Ожидаемый результат

Ответ содержит только безопасные публичные поля: `code`, `message`, `traceId`.

### Влияние

Внутренние детали реализации не должны попадать в ответы для API-потребителей.
