# API/Postman-тестирование

Портфолио-пример тестирования REST API для сервиса заказов интернет-магазина.

В разделе собраны API-сценарии, проверки контрактов, негативные проверки, примеры Postman collection и баг-репорты по типовым дефектам REST API.

## Область тестирования

| Область | Что проверяется |
| --- | --- |
| Авторизация | Авторизация, истечение токена, доступ без токена |
| Каталог | Получение списка товаров, фильтрация, обработка невалидных параметров |
| Корзина | Добавление товара, изменение количества, удаление позиции |
| Заказы | Создание заказа, расчет суммы, статус заказа |
| Негативные проверки | 400/401/403/404/409, невалидные типы данных, отсутствующие поля |
| Контракт ответа | Обязательные поля ответа, типы данных, enum-значения |
| Базовые нефункциональные проверки | Время ответа, идемпотентность, отсутствие 5xx на пользовательских ошибках |

## Подход

- Разделение проверок на позитивные, негативные, контрактные и проверки бизнес-правил.
- Использование переменных окружения: `baseUrl`, `accessToken`, `productId`, `itemId`, `orderId`.
- Сохранение динамических значений из ответов в переменные Postman.
- Проверка JSON schema на ключевых endpoints.
- Проверка, что пользовательские ошибки возвращают 4xx, а не 5xx.

## Пример проверок в Postman

```javascript
pm.test("Статус ответа 201", function () {
  pm.response.to.have.status(201);
});

pm.test("В ответе есть id заказа и корректный статус", function () {
  const json = pm.response.json();
  pm.expect(json.id).to.be.a("string").and.not.empty;
  pm.expect(json.status).to.eql("created");
  pm.collectionVariables.set("orderId", json.id);
});

pm.test("Время ответа в допустимых пределах", function () {
  pm.expect(pm.response.responseTime).to.be.below(1000);
});
```

## Итог

В примере покрыты основные API-риски: неправильная авторизация, некорректный расчет заказа, отсутствие валидации, нестабильные контракты и ошибки обработки негативных сценариев.
