# SQL-проверки данных

Портфолио-пример SQL-проверок для backend/data validation в сервисе заказов.

В разделе собраны SQL-запросы для проверки целостности данных, расчетов по заказам, платежей, статусов и возможных расхождений между сущностями.

## Что проверяется

| Область | Проверка |
| --- | --- |
| Orders | Сумма заказа соответствует сумме позиций |
| Payments | У оплаченных заказов есть успешный платеж |
| Качество данных | Нет дублей пользователей и заказов |
| Целостность | Нет записей в `order_items` без родительского заказа |
| Бизнес-правила | Отмененные заказы не имеют успешных списаний после отмены |
| Аудит | Статусы заказа меняются в допустимой последовательности |

## Схема для примеров

```text
users(id, email, created_at, deleted_at)
orders(id, user_id, status, total_amount, currency, created_at, cancelled_at)
order_items(id, order_id, product_id, quantity, price)
payments(id, order_id, status, amount, created_at)
status_history(id, order_id, old_status, new_status, created_at)
```

## Итог

Набор запросов показывает проверки, которые часто нужны тестировщику при работе с web/API-продуктом: сверка UI/API с базой, поиск неконсистентных данных, расследование багов в заказах и подготовка доказательной базы для баг-репорта.
